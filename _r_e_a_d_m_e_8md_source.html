1 # MFRC522
2 
3 The MFRC522 is a 13.56 MHz contactless reader/writer IC. It supports
4 working with ISO/IEC 14443 A/MIFARE and NTAG. Officially, this chip
5 supports SPI/UART/I2C connectivity to a host microcontroller. However,
6 most cheap MFRC522 modules found online for hobbiests are limited to
7 a single SPI connection.
8 
9 The official MFRC522 [datasheet](https://www.nxp.com/docs/en/data-sheet/MFRC522.pdf)
10 can be found on NXP's website.
11 
12 This is a modern C++ library for the contactless reader/writer MFRC522 chips.
13 This library is agnostic with respect to both the host microcontroller running
14 the code and the library used to drive your SPI peripherals. There are only
15 a few methods needed from your SPI class. Please see below for example SPI
16 implementations.

19 Usage
20 ---------
21 
22 There are two options for including this library with your code:
23 
24 1. You can simply drop MFRC522.cpp and MFRC522.hpp directly with the rest of
25  your source files and add them to your build system. Please see the examples
26  folder to see how to do this.
27 
28 2. You may also use this as a git submodule and integrate it as a CMake project.
29  This method allows you to upgrade this library simply by calling git pull
30  within the submodule folder.

33 Documentation
34 -----------------
35 
36 This library only implements communication with the MFRC522 via a SPI.
37 As stated before, the code does not care how the SPI is implemented as
38 long as it has the following accessible methods:
39 
40 <Class>::write();
41 <Class>::read();
42 <Class>::transfer();
43 <Class>::nssLow();
44 <Class>::nssHigh();
45 <Class>::~Class();
46 
47 These should be pretty simple, but more explanation can be found in the examples
48 directory. It should be noted that the MFRC522 class expresses strong ownership
49 semantics of the SPI peripheral, so your implementation must allow for that.
